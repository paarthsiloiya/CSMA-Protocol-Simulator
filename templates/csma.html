Carrier Sense Multiple Access (CSMA) is a Media Access Control (MAC) protocol used to control how data is transmitted across a shared network medium. It improves upon simple ALOHA protocols by requiring nodes to "listen before they talk."

Purpose & Problem Solved

In shared mediums, if two nodes transmit simultaneously, their signals corrupt each other, causing a collision. Pure ALOHA suffers from frequent collisions. CSMA prevents many collisions by sensing the channel before initiating any transmission.

Working Principle

  1. Sense the Channel: A node with data to send checks if any other node is currently transmitting.
  2. If Idle: The node proceeds to transmit the packet.
  3. If Busy: The node waits (backs off) according to a persistence strategy:
    • 1-persistent: Transmit immediately once idle.
    • Non-persistent: Wait a random time before sensing again.
    • p-persistent: When idle, transmit with probability p.

Strengths and Limitations

Strengths: Simple implementation and significantly reduces collisions compared to ALOHA.
Limitations: Collisions can still occur if two nodes sense the channel as idle simultaneously due to propagation delay. Furthermore, basic CSMA has no mechanism to halt a transmission once a collision happens, leading to wasted bandwidth as the entire corrupted frame is sent.
